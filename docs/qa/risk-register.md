# Risk Register — Weather Explorer

Risks to delivery and to the running app, with likelihood, impact, mitigation,
owner, and status. Status values: **Open**, **Mitigated** (controls in place,
residual risk accepted), **Accepted** (no further action), **Pending-ops**
(needs an ops/deploy action outside this codebase).

Last updated: 2026-06-27 (Europe/Kyiv).

| # | Risk | Likelihood | Impact | Mitigation | Owner | Status |

| R-01 | **Keyless Open-Meteo availability / rate limits.** The app depends on the free, keyless Open-Meteo forecast + geocoding service (NFR-COST-01, TC-STACK-03). An outage or throttle would block forecasts and search. | Medium | High | Every upstream call degrades honestly: route handlers return a typed `{ error: "failed" }` (never a raw 500) and the UI shows a calm Ukrainian message, console stays clean (NFR-OBS-01). Server-side `AbortSignal.timeout` prevents a hung upstream from leaving an infinite spinner (added in the city-search review-gate). In-memory cache holds the last good forecast until the location changes (FR-FORECAST-05). | Eng | Mitigated |
| R-02 | **Nominatim usage-policy compliance.** Reverse-geocoding on map click uses OSM's Nominatim (ADR-0005). Reverse-geocode runs only on an explicit user map click, server-side, not in a loop. Residual: confirm the production deployment sets a contactable User-Agent + correct Referer. | Eng / Ops | Pending-ops |
